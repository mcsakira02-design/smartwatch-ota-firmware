# 系统配置层说明 Config Layer

## 📌 模块定位
该目录用于统一管理整个工程的系统配置项、宏定义、板级信息、软件版本与功能裁剪控制。是实现平台化架构中模块解耦、功能复用的基础。

## 📁 文件说明

| 文件名             | 说明 |
|--------------------|------|
| ProjectConfig.h    | 项目编号、产品型号、调试模式等配置 |
| VersionConfig.h    | 软件版本、编译日期、作者信息等 |
| BoardConfig.h      | 板级资源定义，如IO映射、外设通道编号 |
| FeatureConfig.h    | 功能裁剪宏定义（如 `USE_IMU6050`, `ENABLE_CLI`） |
| RtosConfig.h       | RTOS任务栈大小、优先级、队列长度配置等 |

## 🧩 使用方式
- 各平台层应通过包含对应头文件来读取配置，而不是硬编码
- 统一使用宏开关控制模块启用/禁用

## ✅ 命名规范
- 宏统一以 `CFG_` 前缀，如 `CFG_USE_W25Q128`
- 配置项按模块归类，如 `CFG_UART2_TX_PIN`, `CFG_OS_MAIN_TASK_STACK_SIZE`
